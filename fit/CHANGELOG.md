### sisyphus lane 1 — CLI usability (merged)

First-visitor usability pass. No functional changes to scoring, calibration, or gate logic.

#### Changed

- When log append fails (`OSError`), the CLI exits 2, prints the error on stderr, and does not emit the computed score on stdout (fail-fast for pipelines; interactive users must fix `fit/localonly/` permissions or use `--no-log`).

#### Added

- Repo-root `./corpofit` shell shim → `fit/scripts/corpofit.py`. POSIX; Windows without WSL
  uses `python3 fit/scripts/corpofit.py` directly.
- Mnemonic long-form aliases for the six internal-dimension flags:
  `--psych-safety` (C1), `--wlb` (C2), `--manager` (C3),
  `--security` (C4), `--growth` (C6), `--comp-sustain` (C7).
  The original `--cN` short forms remain fully supported.
- Inline rubric anchors in `--interactive` mode. Before the first
  C1 prompt, the calculator prints the 0 / 30% / 50% / 70% / 90% /
  max scoring band reminders from `fit/docs/role-fit-dimensions.md`,
  and each prompt includes the dimension's human-readable name.
- Profile picker at the top of `--interactive`. Lists calibration
  profiles in `fit/config/profiles/` (sorted by filename). Enter falls
  back to the shipped default; `--config` skips the picker.

#### Removed

- All references to a public industry-classification rubric across
  `fit/README.md`, `fit/scripts/corpofit.py`, `fit/scripts/elicit.py`,
  `fit/docs/getting-started.md`, and `fit/docs/worked-example.md`.
  The 1-10 tier is user-supplied input; corpofit does not ship a rubric.

#### Changed

- `fit/README.md` leads with repo-root `./corpofit` quick start (no Pages hero in lane 1).

## [0.3.2] 2026-05-17

Docs accuracy pass on v0.3.1 user-facing prose.

### Changed

- `docs/PAGES_README.md` History tab description rewritten to match
  what the code actually does. Previous wording said "JSONL score
  log persisted in localStorage with an export-to-JSON button"; in
  reality the browser stores history as a JSON array (localStorage

  the conceptually-parallel file.

No functional changes. CSP, parity tests, LLM wiring, demo page
all unchanged from v0.3.1.

## [0.3.1] 2026-05-17

Bugfix + polish on the v0.3.0 Pages SPA.

### Fixed

- **CSP bug:** `docs/index.html` shipped v0.3.0 with
  `connect-src 'self'`, which blocks every outbound LLM call the
  Phase C/D code is wired for. The v0.3.0 commit message and
  `docs/PAGES_README.md` both incorrectly claimed the CSP allowed
  the managed providers; in reality it did not. The CSP now
  allow-lists `'self'`, `https://api.anthropic.com`,
  `https://api.openai.com`, `https:` (for user-supplied
  openai-compatible URLs over TLS), and `http://localhost:*` plus
  `http://127.0.0.1:*` (so local Ollama / llama.cpp / vLLM work
  without forcing the user to terminate TLS on localhost).

### Changed

- Onboarding profile picker promotes `single-us-metro-2026` to the
  front (it is also the shipped `calibration.example.json` starter
  and the most-common first pick); remaining profiles stay in
  alphabetical order.
- Header gains a "Docs ↗" link pointing to `docs/PAGES_README.md`
  on GitHub.

### Added

- `docs/tests/demo.html`: static snapshot of all three LLM proposal
  panels (Parse JD, Suggest tier with hardblock styling, Elicit
  C1-C7 with mixed score/null/per-dim Accept rows). No live model
  call. Useful for README screenshots and visual review of any
  panel-styling change.
- PAGES_README.md CSP paragraph rewritten to match the actual
  policy and explain why localhost HTTP is allowed.

## [0.3.0] 2026-05-17

Pages SPA. A browser-side port of the scoring logic, plus optional
BYO-LLM assistants for the upstream elicitation work.

### Added

- `docs/index.html` and `docs/assets/` ship a stand-alone single-page
  app. Loads no third-party scripts. Content-Security-Policy meta
  tag restricts `script-src`/`style-src` to `'self'` and
  `connect-src` to the two managed LLM providers plus `https:` for
  user-supplied openai-compatible URLs.
- `docs/assets/score.js`: JavaScript port of `scripts/corpofit.py`'s
  compute step. Tied to the Python implementation by 21 parity
  fixtures under `docs/tests/parity.json`, generated by
  `scripts/gen_parity_fixtures.py` and run by `docs/tests/parity.html`.
- `docs/assets/storage.js`: localStorage abstraction for calibration,
  score log, LLM config, and (separately) LLM API key. The export
  feature reads the log and config but skips the key field.
- `docs/assets/llm.js`: three-provider adapter (Anthropic Claude,
  OpenAI GPT, OpenAI-compatible). Sets the
  `anthropic-dangerous-direct-browser-access` header for Anthropic.
- `docs/assets/prompts.js`: prompt templates for the three
  assistants (parse JD, suggest tier, elicit C1-C7), with a JSON
  parser that tolerates code-fence wrappers and "Here is the JSON:"
  preambles.
- LLM tab: configure provider, model, system prompt, temperature,
  max tokens, and (when needed) API key. "Test connection" sends a
  one-token ping and reports the result or the verbatim error
  including any CORS or auth failure.
- New Job tab: when an LLM is configured, three opt-in assistants
  render above the scoring form. Each surfaces a proposal panel
  with verbatim model output, evidence quotes, and per-field Accept
  buttons. Nothing auto-applies.
- `docs/PAGES_README.md`: walkthrough covering how to enable GitHub
  Pages on this repo, the three privacy invariants, BYO-LLM setup
  per provider (with CORS notes for Ollama and Anthropic), and the
  parity-test workflow.

### Internal

- `docs/_config.yml`: suppresses Jekyll's default theme so the SPA
  serves as raw HTML.
- 21/21 parity fixtures pass at this tag.

## [0.2.0] 2026-05-17

W4 watermark + audit-trail clarity.

### Added

- `scripts/corpofit.py --init`: opt-in bootstrap of `config/calibration.json`
  from the shipped example, injecting a fresh 8-char hex `_install_id`.
  Refuses to overwrite an existing user-override file.
- Calibration JSON now supports an optional top-level `_install_id`
  field. When present, it must be an 8-char lowercase hex string and
  is folded into the canonical revision hash, so two installs with
  identical tunables but different install IDs produce different
  revision values. Default installs ship without one; the field is
  user-opt-in only.

### Changed

- Score-log records: `calibration_hash` field renamed to `_revision`.
  Log `schema_version` bumped from `1.0` to `1.1`. Pre-bump records
  remain readable as-is; the field rename is non-breaking for
  consumers that key off `schema_version`.
- Human-readable CLI output now prints `Calibration: <name> (revision <hash>)`
  instead of `(hash <hash>)`.
- The `Calibration` dataclass exposes `.revision` and `.install_id`
  in place of the prior `.hash` attribute. Script consumers that
  read `.hash` directly will need to update.

### Internal

- `tests/test_calibration_log.py::InstallIdTests`: 8 new tests for
  install_id validation, revision-hash determinism, and bootstrap
  refusal-to-overwrite. 89 tests pass.

## [0.1.0] 2026-05-17

Initial public release.

### Added

- `scripts/corpofit.py`: seven-dimension job-fit calculator with categorical gate enforcement, compensation-floor pre-gate, and append-only local logging.
- `scripts/elicit.py`: elicitation flow for cover-letter prep, resume tailoring, and role-preference capture. Persists session answers locally.
- `config/profiles/`: five calibration profiles for common household structures (single-low-COL, single-US-metro, single-high-COL, couple-no-deps-medium-COL, sole-earner-with-deps-medium-COL).
- `config/calibration.example.json`: shipped default calibration, derived from single-US-metro-2026 profile.
- `docs/`: getting-started guide, role-fit dimension reference, abstract worked example. (The original entry here also listed an "industry-classification rubric"; that doc was never published and the claim is retracted in the Unreleased section above. The 1-10 tier is user-supplied input.)
- `tests/`: unit tests for calculator, calibration loader, elicitation flow, archive round-trip; CLI integration tests via subprocess.
